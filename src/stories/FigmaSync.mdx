# Figma Sync

The token system syncs bidirectionally with Figma via Claude + the Figma Console MCP. The DTCG JSON files in `src/tokens/` are always the source of truth — Figma is a reflection of them, not the other way around (until a designer intentionally pushes a change back).

Each token has its Figma variable ID embedded in `$extensions.figma`, so sync is ID-based and survives renames on either side.

**Prerequisites:**
- Figma Desktop app open with the target file
- Figma Console MCP desktop bridge running
- Claude Code session with the Figma MCP connected

---

## Code → Figma (push tokens to Figma)

Use this when you've updated tokens in `src/tokens/` and want Figma to reflect the changes.

**Step 1 — Build tokens**

```bash
npm run build:tokens
```

**Step 2 — Ask Claude to push**

Open a Claude session with the Figma MCP connected and say:

> "Push all tokens from `src/tokens/` to the Figma variable collections in [file name]. Update any variables that exist and create any that are missing."

Claude will:
1. Read the three DTCG JSON files
2. Convert OKLCH colors → hex, rem → px (Figma-required formats)
3. Call `figma_batch_update_variables` for existing variables, matched by `$extensions.figma.variableId`
4. Call `figma_execute` for alias variables (semantic → base, component → semantic)

**What gets synced:**

| Token type | Figma type | Format |
| --- | --- | --- |
| Color | `COLOR` | Hex (`#0071df`) |
| Dimension | `FLOAT` | Pixels (`16`) |
| Font weight | `FLOAT` | Numeric (`600`) |
| Line height | `FLOAT` | Unitless (`1.5`) |
| Shadow | `STRING` | CSS string |
| Semantic tokens | `VARIABLE_ALIAS` | Live reference to Base variable |
| Component tokens | `VARIABLE_ALIAS` | Live reference to Semantic variable |

## Figma → Code (pull Figma changes back)

Use this when a designer has changed variable values in Figma and you want to bring those changes back into the token JSON.

**Ask Claude:**

> "Read the current variable values from the Figma file and update `src/tokens/` to match any values that have changed. Convert hex colors to OKLCH and px values to rem."

Claude will:
1. Call `figma_get_variables` to read all three collections
2. Diff the values against the current DTCG JSON
3. Write updates back to the appropriate JSON file
4. Report a summary of what changed

**After pulling, always rebuild:**

```bash
npm run build:tokens
```

Then verify in Storybook that nothing looks broken before committing.

## About Figma Variable IDs

Every token in `src/tokens/` carries its Figma variable ID in the `$extensions.figma` field:

```json
{
  "color-blue-500": {
    "$value": "oklch(55% 0.2 250)",
    "$type": "color",
    "$extensions": {
      "figma": {
        "variableId": "VariableID:1:7",
        "collectionId": "VariableCollectionId:1:2"
      }
    }
  }
}
```

Style Dictionary ignores unknown `$extensions` fields, so this has no effect on the token build. Three tokens (`card-shadow`, `dialog-shadow`, `dialog-z-index`) have no Figma counterpart because Figma doesn't support shadow and z-index as variable types — they remain without IDs.

## Migrating to a New Figma File

When the source-of-truth Figma file changes — for example, moving from a sandbox file to a production design system file — all variable IDs change. Because IDs are now embedded in the JSON, follow this order to avoid stale IDs causing confusion during the push:

**Step 1 — Clear the stale IDs**

Ask Claude to:

> "Remove all `$extensions.figma` fields from `src/tokens/base.json`, `src/tokens/semantic.json`, and `src/tokens/component.json`."

This ensures the push to the new file is unambiguously name-based with no risk of old IDs being used for matching.

**Step 2 — Push tokens to the new file**

With the new file open in Figma Desktop and the MCP connected:

> "Create variable collections (Base, Semantic, Component) in [new file name] and push all tokens from `src/tokens/` into them."

**Step 3 — Pull the new variable IDs back**

Once the push is complete, ask Claude to:

> "Read the variable IDs from the new Figma file and write them into `src/tokens/` as `$extensions.figma` on each matching token."

**Step 4 — Update any hardcoded file references**

The file key is visible in the Figma URL: `figma.com/design/<FILE_KEY>/`. Search for the old key:

```bash
grep -r "OLD_FILE_KEY" .
```

Replace it with the new file key anywhere it appears in config or documentation.

**Step 5 — Verify and commit**

```bash
npm run build:tokens
npm run storybook
```

Confirm tokens render correctly in Storybook, then commit the updated JSON files.
